DART Score Design

이직각 프로젝트에 필요한 DART API 조합 설계

이 문서는 OpenDART 공식 가이드를 기준으로, 이직각에서 실제로 붙일 만한 DART API를 “무슨 API를 쓸지 → 어떤 컬럼으로 저장할지 → 어떤 점수에 반영할지”까지 한 번에 설계한 문서다. 핵심 목표는 국민연금 기반 점수의 약점을 보완해 성장성, 안정성, 기업 규모를 더 설득력 있게 만드는 것이다.

설계 원칙

MVP 기준 API 선택

처음부터 모든 DART API를 다 붙이지 않고, 현재 점수 체계에 직접 기여하는 API만 우선 붙인다.

우선 1순위: 기업개황 우선 1순위: 주요계정 우선 1순위: 주요재무지표 2순위: 전체 재무제표 2순위: 회사간 비교 API

점수 반영 원칙

  • 성장성: 매출 성장, 영업이익 성장, 자산 성장, ROE/ROA 개선 등
  • 안정성: 부채비율, 유동비율, 영업현금흐름, 이자보상력 등
  • 채용 활동성: DART보다는 국민연금 데이터 유지
  • 기업 규모: 직원 수 외에 자산·매출 규모를 함께 반영
  • 연봉 추정 신호: DART로 직접 계산하지 않음

점수별 반영안

성장성 35점

  • NPS: 직원 수 증가율, 입사/퇴사 비율 유지
  • DART 추가: 매출 성장률, 영업이익 성장률, 자산 성장률
  • 추천 반영 컬럼: revenue_growth_rate, asset_growth_rate, operating_income

안정성 30점

  • NPS: 이직률, 인원 변동성 유지
  • DART 추가: 부채비율, 유동비율, ROA, 영업현금흐름
  • 추천 반영 컬럼: debt_ratio, current_ratio, operating_cash_flow

채용 활동성 15점

  • 국민연금 데이터 유지
  • DART는 직접 반영하지 않음
  • 이유: 공시 데이터는 채용 시점 민감도와 빈도가 낮음

기업 규모 10점

  • 현재: 직원 수 기반 점수
  • DART 추가: 매출 규모, 자산 규모, 자본 규모
  • 추천 반영 컬럼: revenue, total_assets, total_equity

연봉 추정 신호 10점

  • 국민연금 고지금액 기반 참고 지표 유지
  • DART는 직접 대체하지 않음
  • 이유: DART는 개인 직무·연차별 보상 데이터를 제공하지 않음

리스크 감점

  • NPS: 급감, 탈퇴 사업장 이력 유지
  • DART 확장 시: 자본잠식, 적자 지속, 현금흐름 악화 감점 가능
  • 추천 반영 컬럼: net_income, operating_cash_flow, total_equity
권장 DB 테이블

1. DART 회사 매핑

  • dart_corp_codes: corp_code, corp_name, stock_code, modify_date
  • company_dart_profiles: seq, corp_code, stock_code, corp_cls, induty_code, est_dt, acc_mt
  • 핵심 연결 키: companies.seq ↔ company_dart_profiles.seq

2. 재무 원천 + 계산값

  • company_financial_accounts: 계정성 금액 원천
  • company_financial_metrics: 지표성 계산 결과
  • company_financial_benchmarks: 업종 평균/상대 위치
  • company_financial_statements_raw: 필요 시 원문 보강
실행 순서
1

corp_code 수집

고유번호 API 전체 ZIP을 내려받아 상장사 목록을 저장한다.

2

NPS 회사와 매핑

회사명, 종목코드, 사업자번호가 있으면 가장 정확하게 연결한다.

3

기업개황 적재

상장 여부, 업종, 설립일, 결산월을 프로필 테이블에 넣는다.

4

주요계정 적재

매출, 이익, 자산, 부채, 자본, 현금흐름을 연도별로 적재한다.

5

주요재무지표 적재

ROE, 부채비율, 유동비율, 성장률을 지표 테이블로 적재한다.

6

점수 엔진 반영

성장성·안정성·기업 규모 항목부터 단계적으로 DART 가중치를 추가한다.

이직각 기준 최종 추천안
MVP 조합
고유번호 + 기업개황 + 단일회사 주요계정 조회 + 단일회사 주요 재무지표 조회

이 조합이면 지금 프로젝트에서 가장 부족한 부분인 재무 기반 성장성, 재무 기반 안정성, 직원 수 외 기업 규모 판단을 바로 보강할 수 있다. 반대로 채용 활동성과 연봉 추정 신호는 여전히 국민연금 데이터가 핵심이므로 DART로 대체하지 않는 것이 맞다.

공식 참고 화면: OpenDART 개발가이드의 고유번호, 기업개황, 재무정보조회(단일회사 주요계정/주요 재무지표/재무제표/회사간 비교) 기준으로 설계함.
