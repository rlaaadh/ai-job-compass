이직각 백엔드 구조 설명서

이 문서는 "백엔드를 처음 보는 사람"을 기준으로, 현재 이직각 프로젝트의 서버가 왜 필요한지, 어떤 API를 제공하는지, 각 Python 파일이 무슨 역할을 하는지 쉽게 풀어 설명한 문서입니다.

FastAPI 서버 프레임워크
3개 주요 외부 연동 축
3개 핵심 API 엔드포인트
5개 주요 백엔드 레이어
왜 백엔드가 필요한가

프론트만으로 처리하기 어려운 이유

  • 국민연금 API 키를 브라우저에 직접 노출하면 보안상 위험합니다.
  • 공공 API는 간헐적으로 실패하거나 느려질 수 있어 재시도, 캐시, 예외 처리가 필요합니다.
  • 점수 계산과 비교 로직은 화면이 아니라 서버가 맡는 편이 구조적으로 안정적입니다.
  • AI 해석도 API 키 보호와 응답 형식 통제가 필요하므로 백엔드가 중간에 있어야 합니다.

이 프로젝트에서 백엔드가 하는 일

API 키 보호 국민연금 API 호출 검색 결과 정제 건강도 계산 AI 리포트 생성

프론트는 사용자 화면과 입력을 담당하고, 백엔드는 "외부 데이터 가져오기 + 계산 + 응답 만들기"를 맡습니다.

전체 흐름
1

프론트 요청

사용자가 회사명을 검색하거나, 건강도/비교 분석 버튼을 누릅니다.

2

FastAPI 라우터

어떤 API를 호출했는지 보고 해당 Python 함수로 요청을 보냅니다.

3

NPSClient 호출

국민연금 공공 API에서 원본 데이터를 가져옵니다.

4

정제 + 계산

원본 JSON을 Company 모델로 바꾸고 건강도/이직 추천도를 계산합니다.

5

AI 해석 추가

필요하면 AI가 점수를 "설명"하는 문장을 만듭니다.

6

응답 반환

프론트가 쓰기 쉬운 JSON 형식으로 결과를 돌려줍니다.

API 목록
GET /companies/search?name={회사명}&rows={개수}
역할: 회사명으로 사업장을 검색합니다.
  • 프론트 자동완성과 메인 검색 버튼에 사용됩니다.
  • 2글자 미만 검색은 백엔드에서 바로 빈 배열을 반환합니다.
  • 같은 검색어는 60초 동안 캐시해서 외부 API 흔들림을 줄입니다.
GET /companies/{seq}/health
역할: 특정 회사 하나의 건강도 점수와 AI 해석 리포트를 반환합니다.
  • 회사 상세 페이지에서 사용됩니다.
  • 상세 정보와 월별 통계를 가져온 뒤 점수를 계산합니다.
  • AI가 실패해도 점수 자체는 반환하도록 설계돼 있습니다.
POST /compare
역할: 현재 회사와 관심 회사를 비교해 이직 추천도를 계산합니다.
  • 입력값은 current_seq, target_seq 입니다.
  • 두 회사의 건강도를 각각 계산한 뒤, 그 차이를 기반으로 추천 점수를 냅니다.
  • AI는 "추천 이유 설명"을 만드는 데만 사용합니다.
폴더별 역할
폴더 역할 쉽게 말하면
src/api/ HTTP 요청을 받는 입구 프론트에서 들어온 요청을 어느 함수로 보낼지 결정하는 문
src/pipeline/ 국민연금 API 연동 외부 데이터를 실제로 가져오는 전화기
src/scoring/ 점수 계산 규칙 회사 건강도를 숫자로 계산하는 계산기
src/ai/ AI 해석 리포트 숫자를 사람이 읽기 쉬운 설명문으로 바꾸는 번역기
src/db/ 데이터 모델과 DB 구조 나중에 데이터를 저장할 때 기준이 되는 설계도
주요 Python 파일 설명

src/main.py

백엔드 서버의 시작점입니다.

  • FastAPI 앱을 생성합니다.
  • CORS 설정을 추가합니다.
  • company, comparison 라우터를 연결합니다.

src/api/routers/company.py

회사 검색과 건강도 조회 API가 들어 있습니다.

  • /companies/search 구현
  • /companies/{seq}/health 구현
  • 검색 캐시, 짧은 검색어 차단, 예외 처리 담당

src/api/routers/comparison.py

두 회사를 비교하는 API입니다.

  • 현재 회사와 관심 회사 데이터를 각각 가져옵니다.
  • 건강도 점수를 2번 계산합니다.
  • 최종 이직 추천도와 AI 코멘트를 묶어 응답합니다.

src/api/deps.py

라우터가 공통으로 쓰는 보조 함수 모음입니다.

  • NPS 원본 JSON → Company 객체 변환
  • 월별 통계 JSON → CompanyMonthlyStats 변환
  • 건강도 점수 + AI 리포트 → 최종 응답 JSON 조립

src/pipeline/nps_client.py

국민연금 공공 API 호출 전용 클라이언트입니다.

  • .env에서 NPS_API_KEY를 읽습니다.
  • 회사 검색, 상세 조회, 월별 통계 조회 메서드를 제공합니다.
  • 네트워크 실패 시 1회 재시도합니다.

src/scoring/health_score.py

회사 건강도 계산의 핵심입니다.

  • 성장성, 안정성, 채용 활성도, 규모 적합성, 연봉 신호를 각각 계산합니다.
  • 각 항목을 합산하고 리스크 패널티를 빼서 총점을 만듭니다.
  • AI가 나중에 설명할 수 있도록 breakdown 근거도 같이 만듭니다.

src/scoring/recommendation.py

현재 회사 vs 관심 회사 비교 계산기입니다.

  • 두 회사 총점 차이로 추천 점수를 만듭니다.
  • 강력 추천 / 추천 / 중립 / 비추천 판정을 만듭니다.
  • 연봉 변화 신호는 참고값으로만 제공합니다.

src/ai/report_generator.py

AI가 점수를 설명하는 문장을 만드는 파일입니다.

  • Anthropic SDK를 통해 Claude를 호출합니다.
  • 키가 없거나 실패하면 fallback 텍스트를 반환합니다.
  • 즉, AI가 죽어도 서비스 전체가 죽지 않게 설계돼 있습니다.

src/db/models.py

DB 테이블 설계에 해당하는 ORM 모델입니다.

  • Company, CompanyMonthlyStats, WithdrawnCompany 정의
  • 현재 MVP에서는 DB를 적극 활용하지 않지만, 저장 구조는 미리 준비돼 있습니다.
중요 로직 예시

1. 검색 API 보호 로직

짧은 검색어 차단 + 캐시 사용 + 외부 API 실패 시 fallback

normalized_name = name.strip()
if len(normalized_name) < 2:
    return []

cache_key = (normalized_name, rows)
cached = _search_cache.get(cache_key)
if cached and time.time() - cached[0] <= 60:
    return cached[1]

2. 원본 JSON을 회사 객체로 변환

국민연금 API 응답은 프론트가 쓰기 어려워서 서버에서 먼저 정리합니다.

company = Company(seq=seq)
company.name = raw.get("wkplNm", "")
company.address = raw.get("wkplRoadNmDtlAddr")
company.industry_name = raw.get("vldtVlKrnNm")
company.employee_count = _int(raw.get("jnngpCnt"))

3. 건강도 점수 계산 방식

한 번에 계산하는 게 아니라, 항목별 함수로 나눠 계산합니다.

성장성 35점
안정성 30점
채용 활성도 15점
규모 적합성 10점
연봉 신호 10점
- 리스크 패널티

4. AI는 계산이 아니라 해석만 담당

점수는 코드가 계산하고, AI는 그 숫자가 의미하는 바를 설명합니다.

response = self._client.messages.create(
    model=_MODEL,
    max_tokens=_MAX_TOKENS,
    system=SYSTEM_INSTRUCTION,
    messages=[{"role": "user", "content": prompt}],
)
응답 형식(스키마)

백엔드는 프론트가 쓰기 쉬운 "일정한 JSON 모양"으로 결과를 반환합니다. 이 규칙은 src/api/schemas.py 에 정의돼 있습니다.

  • CompanyBasicResponse: 회사 검색 결과 1개
  • HealthScoreResponse: 회사 건강도 분석 결과
  • CompareRequest: 비교 요청 입력값
  • CompareResponse: 이직 추천 비교 결과
현재 설계의 특징

MVP 친화적

지금은 DB 저장보다 "외부 API 직접 호출 → 계산 → 응답"에 집중한 구조입니다.

실패에 비교적 강함

AI 실패 시 fallback, 검색 시 캐시, 비정상 레코드 무시 등 운영 안정성을 조금씩 넣어두었습니다.

확장 가능

나중에는 DB 캐싱, 배치 수집, 로그인, 통계 대시보드로 확장하기 좋은 구조입니다.

한 줄 요약
이 프로젝트의 백엔드는 단순히 "API를 하나 만드는 서버"가 아니라, 외부 공공데이터를 안전하게 가져오고, 그 데이터를 계산 가능한 구조로 정리한 뒤, 점수와 설명을 만들어 프론트에 전달하는 중간 두뇌 역할을 합니다.
