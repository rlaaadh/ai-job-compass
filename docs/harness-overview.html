주의사항 & 설계 결정

⚠️ 당월고지금액 제약

국민연금 연금 보험료 고지 기준이므로 실제 급여와 다릅니다. 모든 에이전트에서 estimated_salary_signal 등 "참고값" 임을 명시합니다.

⚠️ 탈퇴 사업장 해석

withdrawn_companies 데이터는 "폐업 신호"가 아닌 "리스크 보조 지표"로만 활용합니다. 탈퇴 = 망한 회사가 아닙니다.

ℹ️ AI 역할 분리

AI (claude-haiku)는 점수 계산에 관여하지 않습니다. 점수는 순수 룰 기반 (score-engine), AI는 "왜 이 점수인지 설명"만 담당합니다.

ℹ️ 권장 실행 순서

기획 먼저 → 구현 나중. plan-job-compass로 docs/tech/를 먼저 생성하면 build-job-compass가 이를 참조하여 더 정확하게 구현합니다.

ℹ️ 부분 재실행 지원

기존 산출물이 있는 상태에서 "점수 엔진만 수정해줘"처럼 요청하면 해당 Phase만 재실행합니다. 전체를 처음부터 다시 실행하지 않습니다.

ℹ️ MVP 범위

인증, 캐싱, 레이트 리밋, PostgreSQL 마이그레이션은 MVP 범위 밖입니다. backend 에이전트는 SQLite + 직접 API 호출 방식으로 먼저 구현합니다.
